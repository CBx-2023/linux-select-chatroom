项目简介

本项目实现了一个基于 TCP socket 的多人聊天室。服务端运行在 Linux 终端中，使用单进程 select 同时监听服务端 socket 和多个客户端 socket；客户端同样通过终端输入消息， 支持群聊、私聊、在线用户列表和主动退出。

GitHub Pages 只承载这份使用文档，适合从仓库的 /docs 目录发布。真正的聊天室程序需要在 Linux 环境中编译并运行 code/server 与 code/client。

功能清单

服务端

  • 使用 TCP socket 接受多个客户端连接。
  • 使用 select 管理监听 socket 和客户端 socket。
  • 校验昵称格式，拒绝重复昵称。
  • 转发群聊消息，只发送给其他在线用户。
  • 按昵称投递私聊消息。
  • 返回在线用户列表。
  • 处理 /quit 和异常断开，清理 fd 与昵称状态。
  • 将启动、登录、退出、聊天和错误事件写入 server.log。

客户端

  • 启动后连接服务端并发送 LOGIN <nickname>。
  • 普通文本自动转换为群聊协议 MSG <content>。
  • 支持 /w、/list、/quit 三个终端命令。
  • 将服务端响应格式化为系统通知、群聊、私聊、在线列表、确认或错误信息。
  • 忽略空输入，并在本地拦截不完整的私聊命令。

编译与运行

环境要求

  • Linux 或兼容 POSIX socket 的环境。
  • 支持 C++17 的编译器，例如 g++。
  • make。

编译命令

在仓库根目录执行：

make -C code

该命令会进入 code 目录执行 Makefile，生成 code/server 和 code/client 两个可执行文件。

启动服务端

在 code 目录中启动服务端，监听端口 8888：

cd code
./server 8888

./server 8888 表示让服务端绑定并监听 8888 端口。服务端运行期间保持该终端打开。

启动客户端

另开终端进入 code 目录，连接本机服务端并使用昵称 alice 登录：

cd code
./client 127.0.0.1 8888 alice

./client 127.0.0.1 8888 alice 的三个参数分别是服务端 IP、服务端端口和客户端昵称。 可以再打开其他终端启动 bob、carol 等不同昵称的客户端： 

./client 127.0.0.1 8888 bob
./client 127.0.0.1 8888 carol

客户端命令

终端输入 客户端发送 作用
hello everyone MSG hello everyone 发送群聊消息给其他在线用户。
/w bob hello PRIVATE bob hello 发送私聊消息，只投递给昵称为 bob 的在线用户。
/list LIST 查询当前在线昵称列表。
/quit QUIT 主动退出客户端并通知服务端清理状态。

注意：昵称长度为 1 到 20 个字符，只允许英文字母、数字和下划线。空输入会被忽略； /w 缺少目标昵称或消息正文时，客户端会在本地提示用法并不会发送给服务端。

协议摘要

客户端和服务端使用按行分隔的文本协议，每条协议行以换行结束。单行最大长度为 1024 字节， 超出会得到 ERROR message too long。

客户端请求

协议行 说明
LOGIN <nickname> 连接后登录昵称；登录前只能发送登录或退出请求。
MSG <content> 群聊内容，服务端转发为 CHAT <sender> <content>。
PRIVATE <target> <content> 私聊内容，服务端转发为 PRIVATE <sender> <content>。
LIST 请求在线用户列表。
QUIT 主动退出。

服务端响应

响应前缀 示例 说明
OK OK login successful 操作成功确认。
ERROR ERROR nickname already used 请求格式、登录状态、昵称或投递目标错误。
SYSTEM SYSTEM bob online 用户上下线等系统通知。
CHAT CHAT alice hello 来自其他用户的群聊消息。
PRIVATE PRIVATE alice secret 来自其他用户的私聊消息。
USERLIST USERLIST alice,bob,carol 当前在线用户昵称列表，使用逗号分隔。

手动验收流程

  1. 在仓库根目录执行 make -C code，确认生成 code/server 和 code/client。
  2. 进入 code 目录，执行 ./server 8888 启动服务端。
  3. 分别打开至少三个终端，运行 ./client 127.0.0.1 8888 alice、./client 127.0.0.1 8888 bob 和 ./client 127.0.0.1 8888 carol。
  4. 在一个客户端输入普通文本，确认其他客户端收到群聊消息。
  5. 在 alice 中输入 /w bob hello，确认只有 bob 收到私聊。
  6. 输入 /list，确认返回当前在线用户列表。
  7. 尝试再启动一个昵称同为 alice 的客户端，确认收到 ERROR nickname already used。
  8. 使用 raw TCP 工具发送未登录消息或非法命令，确认服务端返回错误且继续运行。
  9. 发送超过 1024 字节的协议行，确认返回 ERROR message too long 且其他客户端不受影响。
  10. 在一个客户端输入 /quit，确认客户端退出、其他用户收到离线通知。
  11. 直接关闭一个客户端终端，确认服务端清理异常断开连接，其他用户仍可聊天。
  12. 检查 server.log，确认启动、登录、退出、异常断开、群聊、私聊和错误事件有记录。

需要留存验收证据时，可参考 code/validation_checklist.md 逐项记录结果和截图。

常见问题

为什么打开 GitHub Pages 后没有聊天室输入框？

因为 GitHub Pages 只是静态使用文档，不是浏览器版聊天室。项目通信基于 Linux TCP socket， 需要运行 ./server 和 ./client。

make -C code 失败怎么办？

先确认系统安装了 make 和支持 C++17 的 g++。如果编译器版本较旧， 请升级编译器或在课程实验环境中运行。

./server 8888 提示端口不可用怎么办？

可能已有进程占用了 8888 端口。可以停止占用进程，或选择其他端口并让客户端使用相同端口连接。

为什么第二个 alice 登录失败？

服务端要求在线昵称唯一。第二个相同昵称会收到 ERROR nickname already used。

为什么 /w 没有发出去？

/w 必须同时包含目标昵称和消息正文，例如 /w bob hello。 缺少参数时客户端会本地提示 Usage: /w <nickname> <message>。

  • 图形界面、移动端应用或跨平台 UI。
